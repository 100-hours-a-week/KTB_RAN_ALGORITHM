# 문제 보고 알고리즘 감지 루틴

문제를 읽자마자 "무슨 알고리즘을 써야 하는지" 판단하기 위한 실전 체크리스트.

---

## 1) 입력 범위를 먼저 본다

이 단계가 가장 중요하다.

- `N <= 10`  
  완전탐색, 백트래킹, 비트마스크 가능성
- `N <= 1,000`  
  `O(N^2)` 가능
- `N <= 100,000`  
  `O(N log N)`까지 안정권
- `N` 또는 값 범위가 매우 큼 (`1e8`, `1e9`)  
  단순 반복문 위험, 수학/이분탐색/전처리 의심

---

## 2) 무엇을 구하는지 키워드로 분류한다

- "최솟값/최댓값"  
  이분탐색, 그리디, DP 후보
- "최단 거리"  
  BFS/다익스트라 우선
- "연결된 개수"  
  DFS/BFS
- "구간 합 여러 번"  
  누적합
- "K번째 수"  
  정렬, 힙, 값 범위 이분탐색 후보

---

## 3) 데이터 구조를 확인한다

- 배열/수열 중심  
  정렬, 투포인터, 이분탐색
- 그래프(정점/간선)  
  DFS, BFS, 최단경로
- 격자(2차원 맵)  
  BFS/DFS + `dx, dy`
- "시간을 최소/최대" + 단조성  
  파라메트릭 서치

---

## 4) 사고 루틴 (문제 읽자마자)

```txt
입력 범위 확인
-> 허용 시간복잡도 계산
-> 키워드(최소/최단/연결/구간합) 체크
-> 구조(배열/그래프/격자) 파악
-> 후보 알고리즘 1~2개 확정
```

---

## 5) 예시 훈련

```txt
N <= 10
K <= 100,000,000
최솟값 구하기
```

해석:

1. `N`이 작아 완전탐색 가능
2. `K`가 커서 `K` 기준 반복은 위험
3. "최솟값" 키워드 존재

결론:

- 값 범위 이분탐색(파라메트릭 서치) 의심

---

## 6) 실전 훈련법

문제를 풀기 전 30초 루틴:

1. `N, M, K` 범위 먼저 체크
2. 허용 복잡도 계산 (`N^2` 가능한지, `N log N`까지만 되는지)
3. 키워드 반응 (최단, 연결, 구간합, 최솟값/최댓값)
4. 자료구조/그래프/격자 판별

---

## 한 줄 요약

> 알고리즘 선택 능력은  
> `입력 범위 + 키워드 + 구조`를 반복 훈련해서 만든다.
