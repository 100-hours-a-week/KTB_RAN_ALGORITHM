- **문제 제목** : 벽 부수고 이동하기
- **문제 레벨** : Gold III
- **문제 유형** : BFS, 상태 관리(State), 3차원 방문/거리 배열
- **문제 제공** : BOJ 2206
- **문제 링크** : https://www.acmicpc.net/problem/2206
- **코드 파일 (Python)** : (별도 파일 미생성, 아래 코드 블록 기준)

---

## 🧩 문제 설명

`N x M` 격자에서 `(0,0)`에서 `(N-1,M-1)`까지 최단거리로 이동해야 한다.  
단, 이동 중 **벽을 최대 한 번만 부술 수 있다.**

즉, 같은 좌표라도

- 벽을 아직 안 부순 상태
- 이미 한 번 부순 상태

를 서로 다른 상태로 관리해야 한다.

---

## 🧩 문제 핵심 포인트

- 이 문제는 "벽을 어디서 부술지"를 미리 정하는 문제가 아니다.
- 기준 없이 가능한 경우를 탐색하되, BFS로 **짧은 거리부터** 확장하면 된다.
- 핵심은 좌표만 보지 않고 상태를 포함하는 것:
  - 기존: `(x, y)`
  - 변경: `(x, y, broken)`
- `dist[x][y][broken]`를 쓰면 상태별 최단거리를 정확히 분리할 수 있다.

---

## 🧩 주석 기반 인사이트 정리

### 1) "벽 부수는 기준"을 찾으려는 생각에서 벗어나기

- "벽을 부수고도 최단거리가 나온다면 부숴도 된다? 그럼 다 부숴봐야 되는 거 아님?"
- 정답: **특정 기준으로 고르는 게 아니라, 가능하면 상태로 다 탐색**한다.
- 다만 완전탐색이 아니라 BFS라서 짧은 거리 레벨부터 처리된다.

### 2) BFS는 좌표 탐색 + 상태 탐색이다

- 이 문제의 본질은 단순 격자 BFS가 아니라 **상태 그래프 BFS**다.
- 같은 칸 `(x,y)`도 `broken=0`과 `broken=1`은 다른 정점으로 본다.

### 3) "더 짧으면 갱신?" 오해 정리

- 질문: `dist[next]`가 현재 거리보다 크면 갱신해야 하나?
- 정리: BFS 특성상 해당 상태에 **처음 도달한 거리**가 최단거리다.
- 그래서 다익스트라식 재갱신 로직이 필요하지 않다.

### 4) 상태 관리 관점의 새 배열 설계

- 이번 문제의 핵심 수확: "좌표 2차원"에서 끝나지 않고
  **상태 차원을 추가한 3차원 배열 설계**
- `visited[n][m][2]`, `dist[n][m][2]`
- 같은 좌표도 상태가 다르면 방문/거리 기록을 분리해야 한다.

## 🧩 코드 로직 정리

### 전체 흐름

```txt
1. 입력으로 격자 arr를 만든다.
2. visited/dist를 [n][m][2]로 초기화한다.
3. 시작 상태 (0,0,0)을 큐에 넣고 dist=1로 시작한다.
4. BFS 중 (x,y,broken)에서 4방향으로 확장한다.
5. 다음 칸이 길(0)이면 같은 broken 상태로 이동한다.
6. 다음 칸이 벽(1)이고 broken==0이면 벽을 부수고 broken=1로 이동한다.
7. 도착점에 처음 도달하면 해당 dist를 반환한다.
8. 끝까지 못 가면 -1 반환한다.
```

## 🧩 코드 구현 (내 풀이)

```python
import sys
from collections import deque

sys.stdin = open("../예제.txt")
input = sys.stdin.readline

#벽을 부수고도 최단거리가 나온다면 벽을 부숴도된다..? 그럼 다 부셔봐야되는거아님? 벽을 부수는 기준이 있을꺼아니야
#벽을 부수는 기준은 없다. 가능하면 다 해본다 가 핵심
#완전탐색이 아니라 이것도 그래프 탐색이다. 짧은 거리부터
#dist는 벽을 한 번 부술 수 있는 상태를 포함해서 최단거리 BFS를 한다
#기존 (x,y) => (x,y,broken)
#BFS는 단방향이 아니라 옆으로 계속 퍼지면서 이동한다. 감염(확산)의 느낌이 맞다.
#그럼 만약 dist[nextX][nextY]가 현재 거리보다 크면 현재 거리를 넣는 로직도 들어가야되나?
# -> 오해다. 애초에 dist에 처음 들어간 값일 경우가 최단거리이다. 때문에 말자체가 성립되지 않는다.

#아 그러니까 4갈래 길로 만들어지고 BFS는 하나의 직선이 시간선이라고 할때 여러 평행세계가 생겨나는구나. 분기분기분기 가 모두 다르기 때문에
#한방향이 아니라 여러 직선에서 만들어지는 것을 BFS라고 하는 구나. 이걸 이용해서 한 직선에서는 벽한번만 부술수있으니까...와우

#(상태 관리 차원에서 각각 요소가 두가지 상태를 갖고 있음.)벽을 부순 경우 : [False]*2 는 하나의 요소에 두가지 상태를 갖고 있음을 나타냄.


n, m = map(int, input().split())
arr = [list(map(int, input().strip())) for _ in range(n)]

dx = [1,0,0,-1]
dy = [0,1,-1,0]

def bfs():
    visited = [[[False]*2 for _ in range(m)] for _ in range(n)]
    dist = [[[0]*2 for _ in range(m)] for _ in range(n)]

    queue = deque([(0,0,0)])
    visited[0][0][0] = True
    dist[0][0][0] = 1

    while queue:
        x, y, broken = queue.popleft()

        if x == n-1 and y == m-1:
            return dist[x][y][broken]

        for i in range(4):
            nx = x + dx[i]
            ny = y + dy[i]

            if 0 <= nx < n and 0 <= ny < m:
                # 길
                if arr[nx][ny] == 0 and not visited[nx][ny][broken]:
                    visited[nx][ny][broken] = True
                    dist[nx][ny][broken] = dist[x][y][broken] + 1
                    queue.append((nx, ny, broken))

                # 벽
                if arr[nx][ny] == 1 and broken == 0 and not visited[nx][ny][1]:
                    visited[nx][ny][1] = True
                    dist[nx][ny][1] = dist[x][y][0] + 1
                    queue.append((nx, ny, 1))

    return -1

print(bfs())
```

## 🧩 사용된 Python 개념

- `collections.deque` : BFS 큐
- 3차원 리스트 초기화 : `[[[...]*2 for _ in range(m)] for _ in range(n)]`
- 상태 포함 튜플 큐: `(x, y, broken)`
- 상태별 방문 분리: `visited[x][y][0/1]`

---

## 🧩 복잡도

- 시간복잡도 : `O(N*M*2)` → `O(N*M)`
- 공간복잡도 : `O(N*M*2)` → `O(N*M)`

## ⚠️ 오답 포인트

- `(x,y)`만 방문 처리해서 `broken` 상태를 섞어버리는 실수
- 벽 칸 이동 시 `broken` 전환(`0 -> 1`)을 빼먹는 실수
- BFS인데 다익스트라처럼 거리 재갱신 로직을 억지로 넣는 실수
- `visited/dist`를 2차원으로만 만들어 상태 정보를 잃는 실수
