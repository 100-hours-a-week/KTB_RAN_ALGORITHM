* **문제 제목** : 트리의 부모 찾기
* **문제 레벨** : 실버 2
* **문제 유형** : 그래프 탐색, 트리, BFS
* **문제 제공** : BOJ 11725
* **문제 링크** : https://www.acmicpc.net/problem/11725
* **코드 파일 (Python)** : [tree-parent-11725.py](../../algorithm-Python/Search_Graph/tree-parent-11725.py)

---

## 🧩 문제 설명

루트가 `1`인 트리가 주어질 때, 각 노드의 부모 노드를 구하는 문제이다.  
출력은 `2번 노드`부터 `N번 노드`까지의 부모를 한 줄에 하나씩 출력한다.

---

## 🧩 문제 핵심 포인트

* 트리는 사이클이 없고 간선 수가 `N-1`이라, 한 번 방문한 노드의 첫 직전 노드가 부모가 된다.
* 시작점을 `1(루트)`로 고정하고 탐색하면 부모 관계가 자연스럽게 채워진다.
* 부모를 `visitedParentArr[node] = parent` 형태로 저장하면 출력이 단순해진다.

---

## 🧩 내 풀이 방식

* 간선을 양방향 그래프로 만든다.
* `bfs(1)`로 루트부터 탐색한다.
* 방문하지 않은 인접 노드를 만날 때마다:
  * 방문 처리
  * `visitedParentArr[child] = current` 저장
  * 큐에 추가
* 탐색 후 `visitedParentArr[2:]`를 출력한다.

---

## 🧩 코드 로직 정리

### 전체 흐름

```txt
1. n과 n-1개의 간선을 입력받는다.
2. 양방향 인접 구조(graph)를 만든다.
3. visitedParentArr를 0으로 초기화한다.
4. bfs(1) 시작:
   - 방문한 노드의 미방문 자식을 큐에 넣고
   - 자식의 부모를 현재 노드로 기록한다.
5. parent 배열에서 2번 노드부터 출력한다.
```

---

## 🧩 코드 구현 (내 풀이)

```python
import sys
from collections import deque

#1에서부터 시작해서 DFS 탐색으로 찾아내야됨. 
#start = 1이 되는거임 

sys.stdin = open("../예제.txt")
input = sys.stdin.readline

n = int(input())
arr = [];
graph = {}

visitedParentArr = [0]*(n+1);
# print(visitedParentArr)

for _ in range(n-1) :
    arr.append(list(map(int,input().split())))

for u,v in arr :
    if u not in graph :
        graph[u] = [];
    if v not in graph :
        graph[v] = [];
    graph[u].append(v);
    graph[v].append(u);


visited = set();
queue = deque();
# print(graph)


#항상 1에서 시작 
#1에서 시작하여 DFS로 타고 타고 내려가서 2있을때까지 DFS로 내려감.
def bfs(ele) :
    queue.append(ele);
    visited.add(ele);
    
    while queue :
        current = queue.popleft();
        
         
        for child in graph[current] :
            if child not in visited :
                visited.add(child);
                visitedParentArr[child] = current
                queue.append(child);
        
        
start = 1
bfs(start)

print("\n".join(list(map(str,visitedParentArr))[2:]))
```

## 🧩 사용된 Python 개념

* `collections.deque` : BFS 큐 구현 (`popleft`)
* `set` : 방문 여부 체크
* 인접 리스트(딕셔너리) : 양방향 트리 간선 저장
* 인덱스 기반 배열 : 부모 정보 저장 및 순서 출력

## ⚠️ 오답 포인트

* 양방향 그래프에서 방문 체크 없이 탐색하면 부모-자식을 왕복하며 무한 루프가 난다.
* `visitedParentArr` 출력을 `0`번부터 하거나 `1`번까지 포함하면 출력 형식이 틀린다.
* 루트 시작점을 `1`이 아닌 다른 값으로 잡으면 전체 부모 관계가 틀어진다.

---

## 💡 구현 팁 (이번 인사이트 요약)

* 핵심 프레임은 `루트 1에서 시작 -> 처음 도달한 직전 노드를 부모로 기록`이다.
* 노드 번호가 `1..N`으로 연속이면, 부모를 딕셔너리보다 배열로 저장하는 것이 가장 단순하다.
* `visitedParentArr[0]`은 더미, `visitedParentArr[1]`은 루트라서 출력은 `2..N`만 하면 된다.
* 네가 말한 "선형으로 뽑는다"는 감각은 정확하다:
  * 탐색은 그래프 구조로 하고,
  * 결과는 `parent` 배열로 평탄화해서 리스트처럼 출력한다.
* 이 문제는 BFS/DFS 둘 다 가능하고, 본질은 "처음 방문 시 부모 확정"이라는 점이다.
