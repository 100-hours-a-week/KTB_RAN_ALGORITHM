# 오랜만에 이분탐색을 푸는 너에게

> 이분탐색 감이 떨어졌을 때, 문제 풀기 전에 2분만 보고 시작하는 복기 노트.

---

## 1) 먼저, 이 문제가 이분탐색인지 확인한다

아래 신호가 2개 이상 보이면 이분탐색을 먼저 의심하면 된다.

- 정렬된 배열에서 값의 존재 여부/위치를 빠르게 찾는다.
- `N`, `M`이 커서 선형 탐색 `O(N*M)`이 위험하다.
- "최소/최대 가능 값"을 묻고, 조건이 단조적으로 바뀐다.
- 정답 후보 범위를 줄여가며 탐색할 수 있다.

---

## 2) 시작할 때 반드시 세팅하는 4가지

1. `left`, `right` : 현재 탐색 구간 경계
2. `mid` : 중앙 인덱스(또는 값)
3. 비교 기준 : `arr[mid]`와 `target` 또는 `check(mid)` 결과
4. 갱신 규칙 : 찾으면 종료할지, 경계를 더 좁힐지

핵심은 한 줄이다.

```txt
조건이 참/거짓으로 단조면, 정답은 경계에 있다.
```

---

## 3) 구현 뼈대 (Python)

```python
def binary_search(arr, target):
    left, right = 0, len(arr) - 1

    while left <= right:
        mid = (left + right) // 2

        if arr[mid] == target:
            return 1
        if arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1

    return 0
```

## 4) Python 응용: bisect / set

### `bisect`로 존재 여부 찾기

```python
from bisect import bisect_left

def exists_with_bisect(arr, target):
    idx = bisect_left(arr, target)
    return 1 if idx < len(arr) and arr[idx] == target else 0
```

- 전제: `arr`가 정렬되어 있어야 한다.
- 장점: 직접 `left/right` 루프를 안 써도 되어 코드가 짧고 안전하다.
- **bisect_left(arr, target)의 뜻** : target이 들어갈수있는 가장 왼쪽 인덱스
  - 값이 있을 경우 기존에 있던 값은 오른쪽으로 밀리고 현재 값이 가장 왼쪽으로 들어감 예시 : 5(넣을위치) 5(기존위치)
- **bisect_rightt(arr, target)의 뜻** : target이 들어갈수있는 가장 오른쪽 인덱스

### `set`으로 존재 여부 찾기

```python
def exists_with_set(card_set, target):
    return 1 if target in card_set else 0
```

- 전제: `card_set = set(arr)` 전처리
- 장점: 존재 여부 문제에서는 평균 `O(1)` 조회로 가장 빠르다.
- 주의: 정렬 위치/경계(lower/upper bound)가 필요하면 `set`만으로는 부족하다.

### 언제 뭘 쓰면 좋은가

- 단순 존재 여부: `set`
- 정렬 기반 존재 확인 + 경계 확장 가능성: `bisect`
- 경계 이동 감각 연습, 파라메트릭 서치 대비: 직접 이분탐색

---

## 5) 자주 나오는 이분탐색 유형 3가지

1. **존재 여부 탐색**

- 예: 수 찾기, 숫자 카드(10815)

2. **경계 찾기 (lower/upper bound)**

- 예: 숫자 카드2(10816), 중복 개수 계산

3. **파라메트릭 서치**

- 예: 랜선 자르기, 나무 자르기, 입국심사
- `check(x)`가 단조일 때 "처음 true" 또는 "마지막 true"를 찾는다.

---

## 6) 오답이 자주 나는 지점

- 정렬하지 않고 이분탐색을 시작한다.
- `while left <= right` 경계 조건을 헷갈린다.
- `mid` 계산 후 `left/right` 갱신 방향을 반대로 쓴다.
- lower/upper bound에서 `<=`, `<` 조건을 뒤바꾼다.
- 파라메트릭 서치에서 `answer` 갱신 타이밍을 놓친다.
