## 🧩 문제

- **문제 제목** : 플로이드
- **문제 레벨** : Gold IV
- **문제 유형** : 플로이드-워셜, DP(모든 쌍 최단거리), 인접 행렬
- **문제 제공** : BOJ 11404
- **문제 링크** : https://www.acmicpc.net/problem/11404
- **코드 파일 (Python)** : (별도 파일 미생성, 아래 코드 블록 기준)

---

## 🧩 문제 설명

도시 `n`개와 버스 노선 `m`개가 주어진다.  
모든 도시 쌍 `(i, j)`에 대해 `i -> j`로 가는 최소 비용을 출력하는 문제다.  
갈 수 없는 경우는 `0`을 출력한다.

---

## 🧩 문제 핵심 포인트

- 단일 시작점 최단거리가 아니라 **모든 쌍 최단거리** 문제다.
- 플로이드-워셜 핵심 점화식:
  - `dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])`
- `k`는 "중간에 거쳐갈 수 있는 도시 집합"을 확장하는 축이다.
- 같은 출발-도착 노선이 여러 번 들어오므로 입력 시 `min` 처리 필요하다.

---

## 🧩 주석 기반 인사이트 정리

### 1) 3중 for의 의미를 역할로 분리하기

- `for k`: 중간 도시
- `for i`: 시작 도시
- `for j`: 도착 도시

이 순서를 유지해야 "k번 도시까지 거치는 경우"를 단계적으로 누적할 수 있다.

### 2) 상태 관리에서 얻은 배열 설계 인사이트

- 이번 포인트는 좌표 탐색이 아니라 **비용 상태 행렬** 관리다.
- `arr[i][j]`는 "현재까지 알고 있는 i->j 최소 비용" 상태를 저장한다.
- 즉, 문제마다 상태가 바뀌면 배열 구조도 바뀐다.
  - BFS 상태: `(x,y,broken)` 같은 다차원
  - 플로이드 상태: `(출발, 도착)` 2차원 비용 행렬

### 3) "우선 그래프부터 채우고 시작"이 핵심

- 버스 입력 `(a,b,c)`를 먼저 반영해서 기본 행렬을 만든다.
- 이후 3중 루프로 점화식을 적용한다.
- 시작 전에 초기화 2개를 반드시 고정:
  - 자기 자신 비용 `0`
  - 나머지 `INF`

## 🧩 코드 로직 정리

### 전체 흐름

```txt
1. 도시 수 n, 버스 수 m 입력
2. dist 행렬을 INF로 초기화하고 dist[i][i]=0
3. 간선 입력 (a,b,c)을 받아 dist[a][b]=min(dist[a][b], c)
4. k=1..n:
     i=1..n:
       j=1..n:
         dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
5. dist[i][j]가 INF면 0, 아니면 비용 출력
```

### 플로이드 전에 필수 과정

1. `# 자기 자신은 0`
   - `dist[i][i] = 0` 초기화를 먼저 해야 한다.
2. `간선 입력`
   - `dist[a][b] = min(dist[a][b], c)`로 입력 간선을 먼저 반영해야 한다.

위 2개를 끝내고 나서 `k -> i -> j` 플로이드-워셜 3중 루프를 돌린다.

## 🧩 코드 구현 (내 풀이)

```python
import sys

sys.stdin = open("../예제.txt")
input = sys.stdin.readline

#플로이드 워샬
#dist[i][j]=min(dist[i][j],dist[i][k]+dist[k][j])
#for k : 중간 섬
# for i : 시작 섬
#  for j :  끝섬

# 버스의 시작 도시 a, 도착 도시 b, 거리 비용 c

#우선 받은걸 위주로 그래프를 그려야됨.


Num_city = int(input())
Bus_Line_Num = int(input())

arr = [[float('INF')]*(Num_city+1) for _ in range(Num_city+1)]

# 자기 자신은 0
for i in range(1, Num_city+1):
    arr[i][i] = 0

for i in range(Bus_Line_Num) :
    s,e,r = map(int,input().split());
    arr[s][e] = min(arr[s][e], r)


for k in range(1, Num_city+1) :
    for i in range(1, Num_city+1) :
        for j in range(1, Num_city+1) :
            arr[i][j] = min(arr[i][j], arr[i][k] + arr[k][j])

for i in range(1, Num_city+1) :
    for j in range(1, Num_city+1) :
        if arr[i][j] == float('inf') :
             print("0", end=" ")
        else :
            print(arr[i][j], end=" ")
    print()
```

---

## 💡 플로이드-워셜 3중 for 팁

- 루프 순서는 반드시 `k -> i -> j`로 유지한다.
- `k`가 바깥에 있어야 "`1..k` 도시만 경유 허용" 상태가 보존된다.
- `i -> j` 갱신은 "지금까지의 최단거리 vs `k` 경유 거리" 2개만 비교한다.
- 입력에서 중복 간선은 `min`으로 미리 정리해야 점화식이 깨끗해진다.
- 출력 조건:
  - 도달 불가(`INF`)는 `0`
  - 도달 가능은 비용 그대로

---

## 🧩 사용된 Python 개념

- 2차원 리스트 초기화: `[[INF]*(n+1) for _ in range(n+1)]`
- `float('inf')` : 무한대 초기값
- 다중 간선 최소 반영: `dist[a][b] = min(dist[a][b], c)`
- 3중 반복문으로 점화식 수행

- 공간복잡도 : `O(N^2)`

## ⚠️ 오답 포인트

- 루프 순서를 `i-j-k`처럼 바꾸면 점화식 의미가 깨진다.
- `dist[i][i]=0` 초기화를 빼먹으면 자기 자신 비용이 비정상적으로 남는다.
- 중복 간선을 마지막 입력값으로 덮어써서 더 큰 비용이 남을 수 있다.
- 도달 불가를 `INF` 그대로 출력하면 문제 출력 형식과 다르다.
