### 개념
- **깊이 우선 탐색 (Depth-FS)**
	- 최대한 <span color="red">**깊이**</span><span color="red"> </span>내려간 후 더이상 길이 없을 때 옆으로 이동 
	- 루트 노드에서** **다음 분기로 넘어가기 전에** 해당 분기를 완벽하게 탐색**하는 방법
	- 활용 : **미로 **
	- **스택 또는**<span color="red">** 재귀함수 **</span>
<empty-block/>
### ⭐️ 탐색 순서 
1. 발급 받은 키의 인접한 값을 확인한다.
2. 인접한 값이 방문하지 않은 노드라면,
	1. 다시 해당 노드의 인접한 값을 확인한다. (1번 반복)(dfs 재귀)
→ 자연스럽게 백트래킹이 가능함.

### 구현 순서
1. visited 생성
2. DFS 함수 정의
	1. 현재 노드 방문 처리 
	2. <span color="blue">**인접 노드 확인**</span>
	3. **방문 안한 노드는 다시 dfs **
3. 시작 노드에서 DFS 호출 
<empty-block/>
### 구현 순서
> **인접 리스트일때**
```javascript

function dfs(start, graph, visited=new Set()){
    visited.add(start); //맨처음 때문에 여기에 둠
    
    for(const children of graph[start]){
        if(!visited.has(children)){
            // visited.add(start); 여기도 되긴한데 
            dfs(children,graph,visited)
        }
    }

}
```
<empty-block/>
> **인접 행렬일때 **
```javascript
function dfs(start, graph, visited = new Set()) {
    visited.add(start);

    for (let next = 0; next < graph.length; next++) {
        // start와 next가 연결되어 있고
        // 아직 방문하지 않았다면
        if (graph[start][next] === 1 && !visited.has(next)) {
            dfs(next, graph, visited);
        }
    }
}
```
<empty-block/>
### 바뀌는 부분
리스트일 때:
```javascript
for (const child of graph[start])
```
행렬일 때:
```javascript
for (let next=0; next<n ;next++)
```
그리고
```plain text
graph[start][next]===1
```
조건이 추가된다.

### 인접 리스트와 인접 행렬의 차이
인접 리스트는 “연결된 것만 순회”
인접 행렬은 “전체를 돌면서 연결 여부 확인”

### 팁
- 그래프 탐색 문제는 보통 <span underline="true">**인접 리스트**</span> 또는 <span underline="true">**인접 행렬**</span>로 바꿔서 탐색함.
	- 예) \[ \[ 1, 2 \], \[ 2, 5 \], \[ 5, 1 \], \[ 3, 4 \], \[ 4, 6 \] \] 이런식으로 되어있으면 <span color="red">**인접 리스트**</span>로 바꾸는 것을 추천
		- **결과 **
			```javascript
graph = [
  [],
  [2,5],
  [1,5],
  [4],
  [3,6],
  [2,1],
  [4]
]
			```
<empty-block/>
- **인접 리스트**로 바꾸는 이유 : **연결된 것**만 순회하기 때문.
	- 인접 행렬 : 전체를 돌면서 연결 여부를 확인하기 때문에 성능 저하발생
