## 🧩 문제

- **문제 제목** : 쉬운 최단거리
- **문제 레벨** : Silver I
- **문제 유형** : BFS, 2차원 격자 최단거리
- **문제 제공** : BOJ
- **문제 링크** : https://www.acmicpc.net/problem/14940
- **코드 파일** : [easy-shortest-path-14940.js](../../algorithm-JS/Search_Graph/easy-shortest-path-14940.js)

---

## 🧩 문제 설명

`N x M` 지도에서 `2`인 칸이 시작점이다.  
`1`인 칸은 이동 가능한 땅, `0`은 갈 수 없는 칸이다.  
각 칸에서 시작점까지의 최단거리를 출력하고, 원래 `1`인데 도달 불가능한 칸은 `-1`로 출력하면 된다.

---

## 🧩 문제 핵심 포인트

- 시작점(`2`)에서 BFS를 1번 돌리면 모든 도달 가능한 `1`의 최단거리를 동시에 채울 수 있다.
- BFS에서 이웃 칸을 방문할 때 `dist[next] = dist[cur] + 1`로 거리 누적을 한다.
- BFS 후에도 `graph`에 `1`로 남아 있는 칸은 도달 불가능한 칸이므로 `-1` 처리한다.

---

## 🧩 내 풀이 방식

- 먼저 전체 격자를 순회해 시작점(`2`) 좌표를 찾는다.
- 시작점을 큐에 넣고 BFS를 수행하며 `dist` 배열에 거리 값을 기록한다.
- 방문한 땅은 `graph`를 `0`으로 바꿔 재방문을 막는다.
- 마지막으로 `graph`에 남은 `1`을 찾아 `dist`를 `-1`로 바꾼 뒤 출력 문자열로 변환한다.

---

## 🧩 코드 로직 정리

### 전체 흐름

```txt
1. dist 배열을 0으로 초기화한다.
2. graph에서 값이 2인 시작점을 찾는다.
3. 시작점을 queue에 넣고 BFS를 수행한다.
4. BFS 중 이동 가능한 1을 만나면 dist를 현재값+1로 갱신한다.
5. BFS 종료 후 graph에 남은 1은 dist를 -1로 변경한다.
6. dist를 행 단위 문자열로 합쳐 출력한다.
```

---

## 🧩 코드 구현 (내 풀이)

```js
const fs = require("fs"); // /dev/stdin
const input = fs.readFileSync("../예제.txt").toString().trim().split("\n");

//백준 제출 시
// const fs = require("fs");
// const input = fs.readFileSync(0, "utf-8").trim().split("\n");

//BFS로 하는게 나을듯

function bfs(queue, graph, dist, dx, dy, N, M, count) {
  while (queue.length) {
    let [curX, curY] = queue.shift();

    for (let i = 0; i < 4; i++) {
      let nextX = curX + dx[i];
      let nextY = curY + dy[i];

      if (
        nextX >= 0 &&
        nextX < N &&
        nextY >= 0 &&
        nextY < M &&
        graph[nextX][nextY] === 1
      ) {
        dist[nextX][nextY] = dist[curX][curY] + 1;
        graph[nextX][nextY] = 0;
        queue.push([nextX, nextY]);
      }
    }
  }
}

function solution(N, M, graph) {
  const dist = Array.from({ length: N }, () => Array(M).fill(0));
  let startX = 0;
  let startY = 0;
  //2찾기 : 시작점 찾기
  for (let i = 0; i < N; i++) {
    for (let j = 0; j < M; j++) {
      if (graph[i][j] == 2) {
        startX = i;
        startY = j;
      }
    }
  }

  //가야할 곳
  const queue = [];

  let count = 0;

  const dx = [-1, 0, 1, 0];
  const dy = [0, 1, 0, -1];

  queue.push([startX, startY]);
  bfs(queue, graph, dist, dx, dy, N, M, count);

  //못가는 곳 찾기
  for (let i = 0; i < N; i++) {
    for (let j = 0; j < M; j++) {
      if (graph[i][j] == 1) {
        graph[i][j] = 0;
        dist[i][j] = -1;
      }
    }
  }

  return dist.map((i) => i.join(" ")).join("\n");
}

const [N, M] = input[0].split(" ").map(Number);
const arr = input.slice(1).map((i) => i.split(" ").map(Number));

console.log(solution(N, M, arr));
```

---

## 🧩 사용된 JS 개념

- `Array.from({ length: N }, () => Array(M).fill(0))` : 거리 배열 초기화
- `queue.push`, `queue.shift` : BFS 큐 구현
- `map` + `join` : 2차원 결과를 출력 형식 문자열로 변환

---

## 🧩 복잡도

- 시간복잡도 : `O(N*M)` 방문 + `shift()` 누적으로 구현상 최악 `O((N*M)^2)` 가능
- 공간복잡도 : `O(N*M)` (`graph`, `dist`, `queue`)

---

## 🧠 사고 키워드

- 시작점 1개 + 격자 최단거리 = BFS
- 거리 배열은 "현재 칸 + 1" 전파 방식
- BFS 이후 잔여 `1`은 도달 불가 판정

## ⚠️ 오답 포인트

- 시작점(`2`)을 찾지 못한 상태로 BFS를 돌리면 전체 결과가 틀어진다.
- 방문 처리를 하지 않으면 같은 칸이 큐에 여러 번 들어간다.
- 도달 불가능한 `1`을 마지막에 `-1`로 바꾸지 않으면 출력 형식 오답이 된다.
