# 백트래킹 복기: 재귀 호출 트리 시각화

좋아.
이걸 **진짜 트리 구조로 시각화**해줄게.

예시:

```txt
N = 3
M = 2
```

👉 1~3 숫자로 길이 2 수열 만들기  
👉 시작은 반드시 `dfs(0)`

---

## 1️⃣ 재귀 호출 트리 (정상 시작: dfs(0))

```txt
dfs(0)   ← depth = 0, current = []
│
├─ push(1) → dfs(1)   current = [1]
│   │
│   ├─ push(1) → dfs(2)  current = [1,1]  ✅ 출력
│   │        (return)
│   │   pop → current = [1]
│   │
│   ├─ push(2) → dfs(2)  current = [1,2]  ✅ 출력
│   │        (return)
│   │   pop → current = [1]
│   │
│   ├─ push(3) → dfs(2)  current = [1,3]  ✅ 출력
│            (return)
│       pop → current = [1]
│
│   pop → current = []
│
├─ push(2) → dfs(1)   current = [2]
│   │
│   ├─ push(1) → dfs(2)  [2,1]  ✅
│   ├─ push(2) → dfs(2)  [2,2]  ✅
│   ├─ push(3) → dfs(2)  [2,3]  ✅
│
│   pop → []
│
├─ push(3) → dfs(1)   current = [3]
    │
    ├─ push(1) → dfs(2)  [3,1]  ✅
    ├─ push(2) → dfs(2)  [3,2]  ✅
    ├─ push(3) → dfs(2)  [3,3]  ✅
```

---

## 2️⃣ 중요한 포인트 (눈여겨볼 부분)

보면 항상 구조가 이렇다:

```txt
push
  ↓
dfs(다음 depth)
  ↓
return
  ↓
pop
```

👉 pop은 항상 “아래 호출이 끝난 뒤” 실행됨

---

## 3️⃣ depth가 의미하는 것

```txt
depth = 0  → 아직 아무것도 안 고름
depth = 1  → 한 자리 고름
depth = 2  → 두 자리 고름 (완성)
```

그래서 시작은 반드시:

```txt
dfs(0)
```

이어야 함.

---

## 4️⃣ 네가 했던 잘못된 시작: dfs(N)

만약 N=3일 때

```txt
dfs(3)
```

로 시작하면?

```txt
depth = 3
M = 2
```

처음부터 depth !== M

그리고 이미 "3자리 고른 상태"로 시작한 셈이라
논리 구조가 완전히 깨짐.

---

## 5️⃣ 호출 스택 관점으로 보기

실제로 스택은 이렇게 쌓인다:

```txt
dfs(0)
  dfs(1)
    dfs(2)
    ← 여기 끝나야
  pop 실행
← 여기 끝나야
pop 실행
```

👉 재귀는 “계단 내려갔다가 올라오는 구조”

---

## 6️⃣ 초보자가 가장 많이 헷갈리는 지점

❌ “dfs 들어가기 전에 pop이 실행되나요?”

아니.

항상:

```txt
push
  ↓
dfs 끝날 때까지 기다림
  ↓
pop
```

이다.

---

## 7️⃣ 한 문장으로 요약

> 재귀는 내려갈 때 push  
> 돌아올 때 pop
