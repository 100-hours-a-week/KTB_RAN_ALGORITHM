---

## 🧩 문제 설명

정수 `N`이 주어질 때 `N!`의 값에서 **뒤에서부터 연속된 0의 개수**를 구하는 문제다.

---

## 🧩 문제 핵심 포인트

* 뒤의 0 한 개는 `10 = 2 × 5` 한 쌍이 생길 때마다 만들어진다.
* `N!`에서는 `2`의 개수가 `5`보다 항상 많으므로, 답은 `5`의 개수와 같다.
* `25`, `125` 같은 수는 `5`를 여러 개 제공하므로 `N // 5`만 세면 부족하다.

---

## 🧩 내 풀이 방식

* `i = 5`부터 시작해 `i <= N` 동안 반복한다.
* 각 단계마다 `N // i`를 더해 "해당 자릿수의 5 개수"를 누적한다.
* `i *= 5`로 `5, 25, 125, ...`를 순회한다.

---

## 🧩 코드 로직 정리

### 전체 흐름

```txt
1. N 입력
2. count = 0, i = 5
3. while i <= N:
   - count += N // i
   - i *= 5
4. count 출력
```

### 왜 `N // 25`도 더해야 하나?

* `25`의 배수는 `5`를 두 개 가진다.  
  예: `25 = 5 × 5`
* 그래서 `N // 5`로 한 번 카운트된 뒤, `N // 25`로 "추가 1개"를 더해줘야 정확하다.
* 같은 이유로 `125`, `625`도 계속 더한다.

---

## 🧩 코드 구현 (내 풀이)

```python
import sys

# sys.stdin = open("../예제.txt")
input = sys.stdin.readline

n = int(input())

count = 0
i = 5

while i <= n:
    count += n // i
    i *= 5

print(count)
```

---

## 🧩 사용된 Python 개념

* `sys.stdin.readline` : 빠른 입력
* 정수 나눗셈 `//` : 특정 배수의 개수 계산
* `while` 반복 : `5`의 거듭제곱을 순차 탐색

---

## 🧩 복잡도

* 시간복잡도 : `O(log₅N)`
* 공간복잡도 : `O(1)`

## 🔍 트리거 문장

* "`N!`의 뒤에서부터 0의 개수"
* "팩토리얼을 직접 계산하지 않고 구하기"
* "`5`, `25`, `125`의 배수를 고려"

---

## ⚠️ 오답 포인트

* `N // 5`만 계산하고 끝내면 `25` 이상의 중복 `5`를 놓친다.
* `N!`을 직접 계산한 뒤 문자열로 0을 세면 비효율적이다.
* 반복에서 `i += 5`처럼 증가시키면 `25, 125` 보정 로직이 빠져 오답이 된다.
