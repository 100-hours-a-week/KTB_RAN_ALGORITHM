* **문제 제목** : 수 정렬하기 3
* **문제 레벨** : 브론즈 1
* **문제 유형** : 정렬, 카운팅 정렬
* **문제 제공** : BOJ
* **문제 링크** : https://www.acmicpc.net/problem/10989
* **코드 파일** : [sort-number-3-10989.py](../../algorithm-Python/Sort/sort-number-3-10989.py)

---

## 🧩 문제 설명

N개의 수가 주어질 때 오름차순으로 정렬해서 출력하는 문제다.  
입력 개수 N이 매우 크고, 각 수의 범위는 `1 ~ 10,000`으로 제한되어 있다.

---

## 🧩 문제 핵심 포인트

* `arr.sort()` 자체가 문제가 아니라, 입력값 전체를 배열에 보관하는 순간 메모리가 크게 증가한다.
* 이 문제의 핵심은 "중복이 많다"가 아니라 "값의 범위가 작다"이다.
* 값 범위가 `1 ~ 10,000`으로 고정되어 있으므로 인덱스를 값으로 쓰는 빈도 배열(`count`)이 가능하다.

---

## 🧩 내 풀이 방식

* 크기 `10001`의 카운트 배열을 만든다.
* 입력을 받을 때마다 `count[x] += 1`로 빈도만 기록한다.
* `1`부터 `10000`까지 순회하면서, `count[value]`만큼 `value`를 출력한다.
* 출력은 `sys.stdout.write`를 사용해 버퍼링 오버헤드를 줄인다.

---

## 🧩 코드 로직 정리

### 전체 흐름

```txt
1. n 입력
2. count[0..10000] 배열 생성 (모두 0)
3. n번 입력받아 count[값] 증가
4. 1~10000 순회
5. count[value] > 0 이면 value를 count[value]번 출력
```

---

## 🧩 코드 구현 (내 풀이)

```python
import sys

input = sys.stdin.readline

n = int(input())
count = [0] * 10001

for _ in range(n):
    count[int(input())] += 1

write = sys.stdout.write
for value in range(1, 10001):
    if count[value]:
        write((f"{value}\n") * count[value])
```

---

## 🧩 사용된 JS 개념

* `배열 인덱스를 값으로 사용하는 카운팅` : 정렬 대상 자체 대신 분포를 저장
* `입출력 최적화` : `sys.stdin.readline`, `sys.stdout.write`
* `반복문 기반 재구성` : 빈도 배열을 순회하며 정렬 결과 재생성

## ⚠️ 오답 포인트

* `arr = [int(input()) for _ in range(n)]`처럼 전체 데이터를 저장하면 메모리 초과가 난다.
* "중복이 많으니 카운팅 정렬"로 판단하면 기준이 흐려진다. 기준은 중복이 아니라 값의 범위다.
* 출력에서 중첩 루프로 한 줄씩 쓰면 느려질 수 있어, 반복 문자열 출력으로 줄이는 방식이 안정적이다.
