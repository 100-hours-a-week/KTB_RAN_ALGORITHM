## 🧩 문제 설명

정수 `N`개가 주어질 때, 순서를 재배치해서 만들 수 있는 가장 큰 수를 출력하는 문제이다.  
숫자를 그대로 크기 비교하는 것이 아니라, 이어붙인 결과가 더 큰 순서로 정렬해야 한다.

---

## 🧩 문제 핵심 포인트

* 두 수 `x`, `y`의 우선순위는 `x+y`와 `y+x`를 비교해서 결정한다.
* 일반 숫자 정렬이나 첫 자리만 보는 정렬로는 정답을 만들 수 없다.
* 모든 값이 `0`인 경우 `"000..."`이 아니라 `"0"`을 출력해야 한다.

---

## 🧩 내 풀이 방식

* 입력 숫자를 문자열 배열로 받는다.
* `compare(x, y)`에서 `x+y > y+x`면 `x`를 앞에 두도록 `-1`을 반환한다.
* `arr.sort(key=cmp_to_key(compare))`로 커스텀 정렬을 적용한다.
* 정렬 후 첫 원소가 `"0"`이면 `"0"`, 아니면 `join` 결과를 출력한다.

---

## 🧩 코드 로직 정리

### 전체 흐름

```txt
1. N과 숫자 배열을 입력받아 문자열 리스트로 변환한다.
2. compare(x, y)로 x+y와 y+x를 비교한다.
3. cmp_to_key(compare)를 key로 전달해 정렬한다.
4. arr[0]이 "0"이면 0을 출력한다.
5. 아니라면 모든 문자열을 이어붙여 출력한다.
```

---

## 🧩 코드 구현 (내 풀이)

```python
import sys
from functools import cmp_to_key

sys.stdin = open("../예제.txt")
input = sys.stdin.readline

n = int(input())
arr = list(map(str, input().split()))

#앞에 자리가 큰 것부터 정렬
#두번째 자리가 큰 것 부터 정렬

def compare(x,y) :
    if x+y > y+x :
        return -1
    else :
        return 1

arr.sort(key=cmp_to_key(compare))

#앞에 0이 오면 제거
if arr[0] == "0" :
    print("0");
else :
    print("".join(arr))
```

---

## 🧩 사용된 Python 개념

* `functools.cmp_to_key` : comparator 함수를 `sort(key=...)`에서 쓸 수 있는 key 객체로 변환
* `lambda` 함수 : 짧은 key 함수/출력 변환을 한 줄로 표현할 때 사용
* `sort(key=...)` : `key` 인자는 한 개만 받으며, 여러 기준은 `tuple key` 또는 `cmp_to_key`로 처리

## ⚠️ 오답 포인트

* 숫자 내림차순 정렬만 하면 `3`, `30` 같은 케이스에서 오답이 난다.
* `cmp_to_key` 없이 comparator를 `key`에 직접 넣으면 의도대로 동작하지 않는다.
* 정렬 결과가 `"0"`으로 시작할 때 예외 처리하지 않으면 `"000..."` 출력으로 오답이 난다.

## 💡 구현 팁 (이번 인사이트)

* 핵심 비교식은 아래 한 줄이다.

```python
if x + y > y + x:
```

* `sort`에서 comparator를 쓰려면 반드시 `cmp_to_key`로 감싸야 한다.
* Python `sort(key=...)`의 `key`는 한 개만 받는다.
* 여러 조건이 필요하면
  * `key=lambda x: (조건1, 조건2, ...)` 형태의 튜플 키를 쓰거나,
  * 이번 문제처럼 `cmp_to_key(compare)`를 사용한다.
