* 최댓값 갱신

## 🧩 문제 설명

어떤 과학자가 발표한 논문의 인용 횟수를 담은 배열 `citations`가 주어질 때,
이 과학자의 H-Index를 구하는 문제다.

H-Index는 다음 조건을 만족하는 `h`의 최댓값이다.

* 인용 횟수가 `h` 이상인 논문이 `h`편 이상
* 나머지 논문은 인용 횟수가 `h` 이하

---


## 🧩 제한사항

* 과학자가 발표한 논문의 수는 1편 이상 1,000편 이하
* 논문별 인용 횟수는 0회 이상 10,000회 이하

---

## 🧩 입출력 예

| citations | return |
| --- | --- |
| [3, 0, 6, 1, 5] | 3 |

### 입출력 예 설명

* 3편의 논문이 3회 이상 인용되었으므로 H-Index는 3이다.

---

## 🧩 문제 핵심 포인트

* H-Index는 "논문 수"와 "인용 횟수"를 동시에 만족해야 한다.
* 내림차순 정렬 후, i번째(0-index) 논문을 보면 `i + 1`편 기준을 바로 검증할 수 있다.
* `citations[i] >= i + 1`이 마지막으로 성립하는 `i + 1`이 정답이다.

---

## 🧩 내 풀이 방식

코드 기준으로 푼 흐름은 아래와 같음.

* 인용 횟수를 내림차순으로 정렬한다.
* 앞에서부터 보면서 현재 순번(`i + 1`)이 H 조건을 만족하는지 확인한다.
* 만족하면 `answer = i + 1`로 갱신한다.
* 만족하지 않는 순간 이후는 더 작아지므로 반복을 종료한다.

---

## 🧩 코드 로직 정리

### 전체 흐름

```txt
1. citations를 내림차순 정렬
2. i = 0부터 끝까지 순회
3. citations[i] >= i+1 이면 answer 갱신
4. 조건이 깨지면 반복 중단
5. answer 반환
```

---

## 🧩 코드 구현 (내 풀이)

```js
function solution(citations) {
  citations.sort((a, b) => b - a); // 내림차순 정렬

  let answer = 0;

  for (let i = 0; i < citations.length; i++) {
    if (citations[i] >= i + 1) answer = i + 1;
    else break;
  }

  return answer;
}
```

---

## 🧩 사용된 JS 개념

* `sort((a, b) => b - a)` : 숫자 배열 내림차순 정렬
* `for` 반복문 : 순번 기반 조건 검증
* `break` : 조건이 깨지는 시점에서 즉시 종료
