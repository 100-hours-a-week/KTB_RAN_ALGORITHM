### 🧩문제
- 문제 제목 : 성문 열고 닫기
- 문제 유형 : 스택
- 문제 제공 : kyo


| gate     | result  | 설명                            |
| -------- | ------- | ----------------------------- |
| `"<<>>"` | `true`  | 두 번 열고(`<<`) 두 번 정확히 닫힘(`>>`) |
| `"<><>"` | `true`  | 각각 독립적으로 열리고 바로 닫힘            |
| `">><<"` | `false` | 열리기 전에 닫으려 함                  |
| `"<<<>"` | `false` | 닫히지 않은 성문이 남아 있음              |



### 🧩코드 로직
- **내 풀이**
- 하나의 stack에서 성문 관리를 합니다.
- 요소가 < 일 경우 그대로 스택에 쌓입니다.
- 요소가 > 일 경우 스택의 상태에 따라 다르게 진행됩니다.
  1. 스택에 <가 존재할 때 pop을 진행
  2. 스택에 <가 없을 때 **false를 출력**

- 모두 진행 후 배열의 상태에 따라 다르게 진행됩니다.
  1. 배열의 길이가 0일 때 -> **true**
  2. 배열의 길이가 0잉 아닐 때(삭제가 되지 못한 성문 존재) -> **false**

```js

function solution(gate) {
    let stack = [];
    let gateAlpha = [...gate];

    for(const A of gateAlpha){
        //< 이면 stack에 쌓이고
        //>인데 <있으면 사라지고, 없으면 false
        if(A === "<"){
            stack.push(A);
        }else if (A === ">"){
            if(stack.includes("<")){
                stack.pop();
            }else{
                return false;
            }
        }
    }
    return stack.length === 0;
}

```

- **kyo 풀이**
- 요소가 < 일 경우 스택에서 push합니다.
- 요소가 > 일 경우 스택에서 pop합니다.
  - 예외처리 : 스택이 비어있을 경우는 진행할 수 없습니다. 

```js
function solution(gate) {
    const stack = [];

    for (let char of gate){
        if(char === "<"){
            stack.push(char);
        }else{
            if(stack.length === 0){
                return false
            }
            stack.pop();
        }
    }

    return stack.length === 0;
}

```
