### 🧩문제
- 문제 제목 : 명령어 수행하기
- 문제 유형 : 큐
- 문제 제공 : kyo

명령어를 수행한 결과를 배열로 리턴하시오.
- 명령어 종류:
  - ENQUEUE x : 숫자x를 큐에 넣는다.
  - DEQUEUE : 큐에서 맨 앞의 값을 꺼내고 결과 배열에 추가한다. 만약 큐가 비어있으면 "EMPTY"를 결과배열에 추가한다. 


- 입력 형식
    문자열 배열 commands (예: ["ENQUEUE 3","ENQUEUE 5", "DEQUEUE","DEQUEUE","DEQUEUE"] )

- 출력 형식
  DEQUEUE 명령어 수행 결과를 순서대로 담은 배열을 반환

### 🧩코드 로직
- **내 풀이**
- 배열에 들어왔을때 한 요소에서 " "로 잘라서 명령어에 따라 다르게 수행하도록한다.
  - number를 받지 않는 명령어는 어떻게 수행되는지 : undefined가 들어갈뿐 에러사항에 걸리지 않는다. 
- 먼저 ENQUEUE DEQUEUE 수행 방식을 작성한다.
- 이후 EMPTY라는 예외처리 로직을 추가하였다.

```js
function solution(commands) {
    let queue = [];
    let result = [];

    commands.forEach(ele=>{
        let [c,n] = ele.split(" ");
        if(c === "ENQUEUE"){
            queue.push(Number(n));
        }else if(c === "DEQUEUE"){
            let queue_ele = queue.shift();
            let e = queue_ele ? queue_ele : "EMPTY";
            result.push(e);

        }
    })

    return result;
}

```

- **kyo 풀이**
- 비슷했지만 딱 하나 다르다.
- queue에 요소가 들어가있을때 (queue.length) queue에서 요소를 빼고, 아니면 "EMPTY"를 넣는다.

```js
function solution(commands) {
    let queue = [];
    let result = [];

    commands.forEach(command=>{
        let [action,value] = command.split(" ");
        if(action === "ENQUEUE"){
            queue.push(Number(value));
        }else if(action === "DEQUEUE"){
            result.push(queue.length ? queue.shift() : "EMPTY");
        }
    })

    return result;
}
```

### 회고
베이스가 되는 코드를 미리 작성하고 해당 코드에서 예외처리하는 방식이 좋은 것 같다. 
처음부터 어떻게 로직을 짤지 고민하면 시간이 길어지고 시각적으로 보여지지 않아서 생각하는데에 오랜 시간이 걸리는 것 같다. 
그리고, 변수명을 잘 짓는 것에도 연습해야겠다. 단순히 c,n : 이라고 작성하는 것이 아니라 action, value라고 작성하는 것 또한 연습이 필요해보인다.


**- 항상 헷갈리는 forEach vs map**

  **forEach**는 반환값이 없기 때문에 순회하는 것이 목적이고,
  **map**은 반환하기 때문에 새 값으로 변형하기 위한 것에 목적이 있다.
