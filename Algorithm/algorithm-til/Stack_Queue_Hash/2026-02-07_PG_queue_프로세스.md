
## 🧩 문제

* **문제 제목** : 프로세스
* **문제 유형** : 큐 (Queue)
* **문제 제공** : kyo
* 코드 파일 : [프로세스](../../algorithm-JS//1.%20Stack_Queue_Hash/process.js)

### 입력 / 출력 예시

```txt
프로세스: [A, B, C, D]
우선순위: [2, 1, 3, 2]
```

* 숫자가 **클수록 우선순위가 높음**
* 실행 순서는 **우선순위 규칙 + 큐 순서**에 따라 결정됨

---

## 🧩 문제 핵심 포인트

* 단순 정렬 문제가 아님
* **“지금 맨 앞 프로세스보다 더 높은 우선순위가 큐 안에 존재하는가?”** 를 계속 판단해야 함
* 즉,

  * **조회는 전체**
  * **제거/삽입은 앞·뒤**
  * **내가 찾고 싶은 인스턴스를 어떻게 계속 추적할 지?**
* → **Queue + 우선순위 비교 로직** 필요

---

## 🧩 코드 로직 정리 (프로세스 기준)

### 전체 흐름

```txt
1. 큐의 가장 앞 프로세스를 꺼낸다
2. 큐 안에 더 높은 우선순위가 있는지 확인한다
   2-1. 있다 → 다시 큐 맨 뒤에 넣는다
   2-2. 없다 → 실행(제거)한다
3. 큐가 빌 때까지 반복
```

---

## 🧩 코드 구현 예시

```js
function solution(priorities) {
  let queue = priorities.map((priority, index) => ({
    priority,
    index,
  }));

  let order = 0;

  while (queue.length > 0) {
    const current = queue.shift();

    const hasHigherPriority = queue.some(
      (p) => p.priority > current.priority
    );

    if (hasHigherPriority) {
      // 더 높은 우선순위가 있으면 뒤로 보냄
      queue.push(current);
    } else {
      // 실행
      order++;
      // 특정 프로세스의 실행 순서를 묻는 문제라면 여기서 체크
      // if (current.index === location) return order;
    }
  }
}
```

## 🧩 사용된 JS 개념 정리

### `Queue`

* `shift()` : 앞에서 꺼냄
* `push()` : 뒤에 삽입

### `Array.prototype.some()`

```js
queue.some(p => p.priority > current.priority);
```

* 배열 안에 **조건을 만족하는 요소가 하나라도 있으면 `true`**
* 이 문제에서 핵심 판단 로직
