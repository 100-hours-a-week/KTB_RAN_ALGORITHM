# 분할정복 vs DP vs 이분탐색 정리

### 자료구조 분리 질문

```text
1. 범위를 줄이는 문제인가?
2. 문제를 나누는 문제인가?
3. 같은 계산이 반복되는가?
4. 그래프 탐색인가?
```

판단 규칙

```text
한쪽만 버리면 이분탐색
모두 쪼개서 풀면 분할정복
겹치는 계산을 저장하면 DP
연결 관계를 따라가면 BFS/DFS
```

## 알고리즘 자료구조 판별법

- 이분 탐색 : 탐색 범위를 절반씩 줄인다.
- 분할 정복 : 문제를 여러 개의 작은 문제로 나누어 해결한다.
- DP : 반복되는 작은 문제의 결과를 저장하여 재사용한다.

### 질문 1

정렬된 배열에서 값을 찾는 문제인가?

YES -> 이분탐색

특징

```text
정렬된 데이터
값 찾기 / 조건 만족 위치 찾기
범위를 절반으로 줄임
```

예시

```text
특정 값 찾기
lower bound
upper bound
k번째 수
```

시간복잡도

```text
O(log N)
```

구조

```text
left / right / mid
```

### 질문 2

문제를 여러 개로 나눠서 각각 해결해야 하는가?

YES -> 분할정복

특징

```text
큰 문제 -> 작은 문제 여러 개
재귀 호출
부분 문제 서로 겹치지 않음
```

예시

```text
색종이
쿼드트리
merge sort
Z-order
```

구조

```text
mid 기준으로 영역 분할
```

재귀 트리 예

```text
n
├ n/2
└ n/2
```

### 질문 3

같은 계산이 반복되는가?

YES -> DP

특징

```text
부분 문제가 겹침
memoization 또는 table 사용
```

예시

```text
피보나치
LIS
배낭
동전 문제
```

재귀 트리 예

```text
f(5)
├ f(4)
│ ├ f(3)
│ └ f(2)
└ f(3)
```

여기서 `f(3)`이 여러 번 등장한다.

### 시험용 핵심 비교

| 질문                    | 알고리즘         |
| ----------------------- | ---------------- |
| 정렬된 배열에서 값 찾기 | Binary Search    |
| 문제를 여러 개로 나눔   | Divide & Conquer |
| 같은 계산 반복          | DP               |

### 문제 키워드 기반 감지

| 문제 키워드               | 알고리즘 |
| ------------------------- | -------- |
| 정렬된 배열               | 이분탐색 |
| 절반 / mid / 영역         | 분할정복 |
| 최댓값 / 최소 / 경우의 수 | DP       |
| 연결 / 이동               | BFS DFS  |

### 지금 푼 문제 기준

색종이

```text
영역 분할
4개로 나눔
```

-> 분할정복

LIS

```text
부분 문제 반복
```

-> DP

이 4개로 대부분의 유형이 정리된다.

### 학습 흐름 평가

```text
그래프
-> 분할정복
-> DP
-> 이분탐색
```

구조 기준으로 연결해서 이해하고 있는 흐름이다.

## 분할정복 vs DP vs 이분탐색 정리

이 문서는 다음 세 가지를 헷갈리지 않게 구분하기 위한 정리이다.

- Binary Search: 범위를 줄이는 탐색
- Divide & Conquer: 문제를 쪼개서 해결하고 결합
- Dynamic Programming: 겹치는 부분 문제를 저장해서 재사용

---

## 1. 분할정복 정의

분할정복은 3단계로 설명할 수 있다.

```text
1. Divide (분할)
2. Conquer (정복)
3. Combine (결합)
```

### Divide

큰 문제를 같은 형태의 작은 문제로 나눈다.

### Conquer

작아진 문제를 재귀적으로 해결한다.

### Combine

작은 문제의 결과를 합쳐 큰 문제의 답을 만든다.

### 결합 예시

| 알고리즘   | 결합 방식         |
| ---------- | ----------------- |
| Merge Sort | 정렬된 배열 merge |
| 색종이     | 개수(count) 합산  |
| 쿼드트리   | 문자열 결합       |

---

## 2. 재귀 트리 관점

분할정복을 재귀 트리로 보면 구조가 더 명확하다.

```text
N
├ N/2
├ N/2
├ N/2
└ N/2
```

핵심은 범위를 기준으로 자식 문제를 만들고, 각 자식을 다시 같은 규칙으로 처리하는 점이다.

---

## 3. 분할정복 vs DP

문장 자체는 맞다.

> DP도 작은 문제로 큰 문제를 푼다.

하지만 핵심 차이는 "부분 문제의 중복"이다.

| 구분      | 분할정복           | DP                     |
| --------- | ------------------ | ---------------------- |
| 부분 문제 | 대체로 겹치지 않음 | 겹침                   |
| 저장      | 보통 불필요        | memoization/table 필요 |
| 목적      | 문제 분할 후 해결  | 중복 계산 제거         |

### 분할정복 예시

```text
merge sort
```

같은 부분 배열을 다시 계산하지 않는다.

### DP 예시

```text
fibonacci
fib(3)
fib(3)
fib(3)
```

같은 문제가 반복 호출되므로 저장이 필요하다.

---

## 4. 분할정복과 Top-down 관계

"분할정복은 탑다운인가"라는 질문은 반만 맞다.

- 맞는 이유: 재귀 호출 흐름이 보통 `큰 문제 -> 작은 문제`로 내려간다.
- 정확한 정의: Top-down은 계산 방향, Divide & Conquer는 문제 분할 전략이다.

| 개념             | 의미           |
| ---------------- | -------------- |
| Top-down         | 계산 방향      |
| Divide & Conquer | 문제 분할 방식 |

즉, 비슷하게 보일 수 있지만 같은 개념은 아니다.

---

## 5. 왜 이분탐색과 비슷하게 느껴지는가

공통으로 `mid`를 자주 사용하기 때문이다.

| 알고리즘      | mid 사용 방식             |
| ------------- | ------------------------- |
| Binary Search | 탐색 범위를 절반으로 줄임 |
| Merge Sort    | 배열 분할                 |
| Segment Tree  | 구간 분할                 |
| 분할정복 문제 | 영역 분할                 |

그래서 체감상 "이분탐색 같다"고 느끼는 것은 자연스럽다.

---

## 6. 이분탐색과 분할정복의 실제 차이

### Binary Search

```text
한쪽만 탐색

n
└ n/2
  └ n/4
```

### Divide & Conquer

```text
모든 자식 문제 해결

n
├ n/2
└ n/2
```

차이는 "분할 후 몇 개를 실제로 푸는가"이다.

---

## 7. 코테에서 분할정복을 만났을 때 체크할 것

```text
1. 범위
2. 분할 기준 (mid)
3. 종료 조건
```

예:

```javascript
mid = (start + end) / 2;
```

여기에 "결합 방식"까지 붙이면 실전 설계가 완성된다.

```text
4. 결합 방식 (합산, merge, 문자열 결합 등)
```

---

## 8. 정리 노트 버전

```text
분할정복

1. Divide
큰 문제를 같은 형태의 작은 문제로 분할한다.

2. Conquer
작은 문제를 재귀적으로 해결한다.

3. Combine
작은 문제의 결과를 결합하여 큰 문제를 해결한다.

특징
- 재귀 트리 구조
- 범위를 기준으로 문제 분할
- 부분 문제는 서로 겹치지 않는다.

DP와 차이
- DP: 부분 문제가 겹친다.
- Divide & Conquer: 부분 문제가 대체로 겹치지 않는다.

문제 접근
1. 범위
2. 분할 기준(mid)
3. 종료 조건
4. 결합 방식
```

## 9. 알고리즘별 요약

### Binary Search

```text
정렬된 데이터에서 범위를 절반씩 줄이며 탐색한다.
매 단계에서 한쪽 구간만 탐색한다.
```

특징

```text
한 번 비교할 때 탐색 범위가 1/2로 감소
시간복잡도 O(log N)
```

구조

```text
n -> n/2 -> n/4 -> n/8
```

### Divide & Conquer

```text
큰 문제를 같은 형태의 작은 문제로 분할하고,
각 문제를 해결한 뒤 결과를 결합한다.
```

특징

```text
문제를 여러 개로 나눠 모두 해결
부분 문제는 대부분 겹치지 않음
재귀 트리 구조
```

예시

```text
Merge Sort
Quick Sort
색종이 문제
쿼드트리
```

### Dynamic Programming

```text
큰 문제를 작은 문제로 나누어 해결하되,
겹치는 부분 문제 결과를 저장해서 재사용한다.
```

특징

```text
부분 문제 중복 발생
memoization 또는 table 사용
```

예시

```text
피보나치
LIS
동전 문제
배낭 문제
```

## 10. 핵심 비교표

| 알고리즘         | 핵심 특징                               |
| ---------------- | --------------------------------------- |
| Binary Search    | 탐색 범위를 절반씩 줄인다               |
| Divide & Conquer | 문제를 여러 개로 나눠 해결하고 결합한다 |
| DP               | 같은 부분 문제 계산을 저장해 재사용한다 |

## 11. 구조 차이 요약

### 이분탐색

```text
n
└ n/2
  └ n/4
```

한쪽만 내려간다.

### 분할정복

```text
n
├ n/2
└ n/2
```

분할된 자식들을 모두 해결한다.

### DP 재귀 트리 예

```text
n
├ n-1
│  ├ n-2
│  └ n-3
└ n-2
```

같은 문제가 겹쳐서 등장하므로 저장이 필요하다.

## 12. 이 정리에서 좋은 포인트

다음 문장은 매우 중요한 관찰이다.

```text
범위가 겹치는 구간이 있어서 메모이제이션을 사용한다.
```

이 문장이 DP의 핵심을 정확히 짚는다.

## 13. 한 줄 정리

```text

```

---

## 14. 실전 3초 판별 체크

### Binary Search 체크

```text
- 정렬 또는 단조성(가능/불가능 경계)이 보인다
- 매 단계에서 한쪽 구간만 버린다
```

### Divide & Conquer 체크

```text
- 같은 형태의 하위 문제로 분할 가능하다
- 분할한 자식 문제를 모두 풀어야 한다
- 분할 기준(mid/구간/좌표)이 명확하다
```

### DP 체크

```text
- 동일한 부분 문제가 반복 등장한다
- 상태 정의(dp[i], dp[i][j])가 가능하다
- 점화식 + 베이스 케이스를 세울 수 있다
```
