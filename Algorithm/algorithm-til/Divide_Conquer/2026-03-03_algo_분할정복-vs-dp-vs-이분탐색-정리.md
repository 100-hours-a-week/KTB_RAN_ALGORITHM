# 1️⃣ 분할정복 정의 (정확한 버전)

분할정복은 **3단계 구조**입니다.

```text
1. Divide (분할)
2. Conquer (정복)
3. Combine (결합)
```

### 1. 분할

큰 문제를 **같은 형태의 작은 문제**로 나눈다.

### 2. 정복

작아진 문제를 **재귀적으로 해결한다.**

### 3. 결합

작은 문제의 결과를 **합쳐서 큰 문제의 답을 만든다.**

예

| 알고리즘       | 결합       |
| ---------- | -------- |
| Merge Sort | 배열 merge |
| 색종이        | count    |
| 쿼드트리       | 문자열 결합   |

---

# 2️⃣ 네 정리 중 좋은 부분

이 부분은 **아주 정확합니다.**

```text
분할정복은 재귀트리 구조이다.
범위별로 나눠서 자식을 만든다.
```

실제로 분할정복은 **Recursion Tree**로 설명됩니다.

예 (이번 문제)

```text
N
├ N/2
├ N/2
├ N/2
└ N/2
```

---

# 3️⃣ 분할정복 vs DP (여기만 조금 수정)

너의 문장

> DP도 작은 문제로 큰 문제를 푼다

✔ 맞습니다.

하지만 **핵심 차이가 하나 있습니다.**

| 구분    | 분할정복       | DP             |
| ----- | ---------- | -------------- |
| 작은 문제 | **겹치지 않음** | **겹침**         |
| 저장    | 필요 없음      | memoization 필요 |
| 목적    | 문제 분할      | 중복 제거          |

예

### 분할정복

```text
merge sort
```

같은 부분 배열을 **다시 계산하지 않음**

---

### DP

```text
fibonacci
```

같은 문제가 계속 등장

```text
fib(3)
fib(3)
fib(3)
```

그래서 **memoization 사용**

# 4️⃣ 분할정복 = 탑다운?

이건 **반은 맞고 반은 틀립니다.**

### 맞는 이유

보통 재귀라서

```text
큰 문제 → 작은 문제
```

형태입니다.

그래서 **Top-down처럼 보입니다.**

---

### 하지만 정확한 정의

| 개념               | 의미       |
| ---------------- | -------- |
| Top-down         | 계산 방향    |
| Divide & Conquer | 문제 분할 방식 |

즉 **다른 개념입니다.**

# 5️⃣ 왜 이분탐색이 비슷하게 느껴질까

이건 아주 좋은 질문입니다.

이유

```text
mid
```

를 사용하기 때문입니다.

공통 구조

| 알고리즘          | mid 사용 |
| ------------- | ------ |
| Binary Search | 범위 절반  |
| Merge Sort    | 배열 분할  |
| Segment Tree  | 구간 분할  |
| 분할정복 문제       | 영역 분할  |

그래서 체감상

```text
이분탐색 같음
```

이라는 느낌이 드는 게 정상입니다.

# 8️⃣ 최종 정리 (너의 노트 정리 버전)

정리하면 이렇게 쓰면 좋습니다.

```text
분할정복

1. Divide
큰 문제를 같은 형태의 작은 문제로 분할한다.

2. Conquer
작은 문제를 재귀적으로 해결한다.

3. Combine
작은 문제의 결과를 결합하여 큰 문제를 해결한다.

특징
- 재귀 트리 구조
- 범위를 기준으로 문제 분할
- 부분 문제는 서로 겹치지 않는다.

DP와 차이
- DP : 부분 문제가 겹친다.
- Divide & Conquer : 부분 문제가 겹치지 않는다.

문제 접근 시 생각할 것
1. 범위
2. 분할 기준 (mid)
3. 종료 조건
```

# 네 정리 → 조금만 다듬은 버전

### 이분 탐색 (Binary Search)

```text
정렬된 데이터에서 탐색 범위를 절반씩 줄이며 탐색한다.
매 단계마다 한쪽 구간만 탐색한다.
```

특징

```text
한 번 비교 → 탐색 범위 1/2 감소
시간복잡도 O(log N)
```

구조

```text
n
→ n/2
→ n/4
→ n/8
```

즉 **한 방향으로 내려가는 구조**

---

# 분할 정복 (Divide & Conquer)

```text
큰 문제를 같은 형태의 작은 문제로 분할하고
각 작은 문제를 해결한 뒤 결과를 결합하여 큰 문제를 해결한다.
```

특징

```text
문제를 여러 개로 나눠서 모두 해결
부분 문제는 대부분 서로 겹치지 않는다
재귀 트리 구조
```

구조 예

```text
n
├ n/2
└ n/2
```

예시

```text
Merge Sort
Quick Sort
색종이 문제
쿼드트리
```

---

# DP (Dynamic Programming)

```text
큰 문제를 작은 문제로 나누어 해결한다.
하지만 작은 문제들이 서로 겹치기 때문에
이미 계산한 결과를 저장(memoization)하여 재사용한다.
```

특징

```text
부분 문제가 겹침
memoization 또는 table 사용
```

예

```text
피보나치
LIS
동전 문제
배낭 문제
```

---

# 핵심 차이 (객관적 비교)

| 알고리즘 | 특징              |
| ---- | --------------- |
| 이분탐색 | 탐색 범위를 절반씩 줄임   |
| 분할정복 | 문제를 여러 개로 분할    |
| DP   | 같은 문제 반복 계산을 저장 |

# 구조 차이

### 이분 탐색

```text
n
└ n/2
  └ n/4
```

한쪽만 탐색

---

### 분할 정복

```text
n
├ n/2
└ n/2
```

모든 자식 문제 해결

---

### DP (재귀 트리)

```text
n
├ n-1
│  ├ n-2
│  └ n-3
└ n-2
```

문제가 겹침 → memoization

이 문장은 아주 좋습니다.

```text
범위가 겹치는 구간이 있어서 메모이제이션을 사용한다.
```

이게 **DP의 핵심 정의**입니다.
