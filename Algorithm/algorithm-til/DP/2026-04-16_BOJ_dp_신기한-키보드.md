## 🧩 문제

- **문제 제목** : 신기한 키보드
- **문제 레벨** : Gold III
- **문제 유형** : DP, 문자열 인덱스 전처리, 구간 이동 비용
- **문제 제공** : BOJ 1796
- **문제 링크** : https://www.acmicpc.net/problem/1796
- **코드 파일 (Python)** : (별도 파일 미생성, 아래 코드 블록 기준)

---

## 🧩 문제 설명

문자열 `S`의 문자들을 알파벳 순서대로 처리하면서,  
커서를 이동하고 출력(입력) 키를 눌러 모든 문자를 출력할 때의 최소 키 사용 횟수를 구하는 문제이다.

핵심은 문자 하나씩 순차 처리하는 시뮬레이션이 아니라,  
**같은 알파벳 묶음을 한 번에 처리하는 DP 최적화 문제**로 보는 것이다.

---

## 🧩 문제 핵심 포인트

- BFS처럼 "최단 경로"를 찾는 문제가 아니라, 상태를 누적하는 "최솟값 DP" 문제다.
- 알파벳별 등장 위치를 그룹으로 묶어 처리한다.
- 각 알파벳 단계마다 끝나는 위치는 큰 분기 2개다.
  - 해당 알파벳 구간의 왼쪽 끝에서 끝나는 경우
  - 해당 알파벳 구간의 오른쪽 끝에서 끝나는 경우
- 점화식 구조는 일관되게 `과거 최소 비용 + 현재 선택 비용`이다.

---

## 🧩 내 주석 사고흐름 정리 (빠짐없이)

### 1) DP를 어떻게 정의했는가

- `1. DP 정의 : 키 최소사용 횟수 : dp[i]`
- `2. i번째에서 내가 선택할 수 있는 선택 -> 왼쪽, 오른쪽`
- `3. 현재를 정의한 후 이전 상태를 작성한다.`
- `과거 중의 최솟값 + 현재 선택(2)`

### 2) 왜 BFS가 아니라 DP인가

- `BFS 인줄알았는데 DP 인 이유`
- `최단거리 vs 최솟값`

### 3) 처리 단위에 대한 핵심 전환

- `일단 그룹으로 묶어 각 알파벳 위치를 배열에 저장`
- `문자를 하나씩 처리하는 문제가 아니라, 알파벳 단위로 처리하는 문제`
- `알파벳 순서에 맞게 정렬`
- `알파벳 순서대로 출력하고 싶음`
- `커서가 있는 곳만 출력하고, 출력했을때 알파벳은 빈공백으로 바뀜.`

### 4) 상태 분기 아이디어

- `dp[i] = i번째에서 내가 할 수 있는 선택 (왼쪽으로 갈꺼냐 오른쪽으로 갈꺼냐)`
- `AABACA 가 있을때 A를 어디서부터 시작하냐(왼쪽/오른쪽)로 이동 거리가 달라짐`
- `dp[i][0] : i의 알파벳의 커서를 가장 왼쪽에서 시작했을때의 총 거리 계산`
- `dp[i][1] : i의 알파벳의 커서를 가장 오른쪽에서 시작했을때의 총 거리 계산`

### 5) 상태/위치 저장을 분리한 관점

- `dp는 총 이동한 비용을 계산하는 배열`
- `현재 위치를 저장하는 배열은 따로 존재`
- `arr = { A:[0,1], B:[2,3]}`
- `letters 배열의 인덱스가 알파벳 처리 순서`
- `i의 객체를 찾아서 배열의 모든 요소를 순회해야한다. [0,1] 탐색`

### 6) 전이식을 만들 때의 메모

- `dp[B][0] = (1)dp[A][0] + (2)이전 최종 위치에서 현재 시작 위치 이동 + (3)현재 알파벳 구간 내부 이동 + (4)출력 횟수`
- `prev_pos_left = arr[A][len(arr[A])-1]` 같은 "이전 단계의 실제 끝 위치"가 필요
- `골드 DP는 점화식 + 보조 계산 요소(거리/횟수) 정리가 함께 필요`라는 인사이트

### 7) 시뮬레이션과 DP의 경계 정리

- `지우는 것까지 포함이라면 출력으로 인식`
- `str = aaa -> str[0] = " " 같은 직접 수정이 필요한가?`
- `안한다고 함. DP는 계산문제지 시뮬레이션이 아니다.`
- `DP에서는 절대 직접 수정하는 일, 계산 외에는 하지 않는다.`

## 🧩 코드 로직 정리

### 전체 흐름

```txt
1. 문자열 S를 입력받고, 알파벳별 등장 위치 배열(indexArr)을 만든다.
2. 등장한 알파벳만 뽑아 letters를 정렬한다.
3. dp[i][0], dp[i][1]를 준비한다.
   - i번째 알파벳을 처리하고 왼쪽/오른쪽 끝에서 끝나는 최소 비용
4. 첫 알파벳을 기준으로 base case를 채운다.
   - 시작 커서(0)에서 왼쪽/오른쪽 끝까지 이동
   - 해당 알파벳 구간 내부 이동 거리
   - 출력 횟수(등장 개수)
5. i=1..L-1 순회하며 점화식으로 갱신한다.
   - 이전 단계의 왼쪽/오른쪽 끝 위치에서 현재 구간 시작점으로 이동
   - 현재 구간 내부 이동 + 출력 횟수 추가
6. 마지막 알파벳 처리 후 min(dp[-1][0], dp[-1][1]) 출력
```

### 상태 의미 (구현 기준)

- `dp[i][0]` : `i`번째 알파벳까지 처리했을 때 최소 비용 (구현상 한쪽 끝 상태)
- `dp[i][1]` : `i`번째 알파벳까지 처리했을 때 최소 비용 (반대쪽 끝 상태)

구현에서는 `prev_left`, `prev_right`를 함께 갱신하며 전이한다.

## 🧩 코드 구현 (내 풀이)

```python
import sys
from collections import defaultdict

sys.stdin = open("../예제.txt")
input = sys.stdin.readline



#1. 문자열 받아서 위치 배열 만들기

S = input().strip()

arr = defaultdict(list)

for i,v in enumerate(S) :
    arr[v].append(i)

indexArr = dict(arr)


letters = sorted(indexArr.keys())




#2. #dp[B][0] = (1)dp[A][0](전의 A를 처리한 비용) + (2)abs(전의 최종 거리(1일경우) - 현재 B의 거리()) + (3)B의 모든 요소를 이동했을 때의 거리 + (4)출력을 누른 횟수
#(2): dp[A] : 가장 오른쪽에서 끝났으니가, i_right

dp = [ [0]*2 for i in range(len(letters))]

first = letters[0]

i_left = indexArr[first][0]
i_right = indexArr[first][len(indexArr[first])-1]

prev_left = indexArr[first][0]
prev_right = indexArr[first][len(indexArr[first])-1]

i_dict = abs(i_left - i_right)
printCount = len(indexArr[first])

dp[0][0] = abs(0-i_left) + i_dict + printCount
dp[0][1] = abs(0-i_right) + i_dict + printCount


for i in range(1,len(letters)) :
    ch = letters[i]

    #ch의 가장 왼쪽 위치 인덱스 : i_left
    i_left = indexArr[ch][0]
    #ch의 가장 오른쪽 위치 인덱스 : i_right
    i_right = indexArr[ch][len(indexArr[ch])-1]

    i_dict = abs(i_left - i_right)
    printCount = len(indexArr[ch])

    #dp[i][0] : 오른쪽 끝에서 끝남.-> 오른쪽에서 끝났으니까 prev_right에 최종 위치겠고, 이번 인덱스를 가장 왼쪽에서 오른쪽으로 이동시킬꺼라 i_left를 빼야됨.
    dp[i][0] = min(dp[i-1][0] + abs(prev_right - i_left) + i_dict + printCount,
                   dp[i-1][1] + abs(prev_left - i_left) + i_dict + printCount)
    dp[i][1] = min(dp[i-1][0] + abs(prev_right - i_right) + i_dict + printCount,
                   dp[i-1][1] + abs(prev_left - i_right) + i_dict + printCount)

    prev_left = indexArr[ch][0]
    prev_right = indexArr[ch][len(indexArr[ch])-1]


print(min(*dp[-1]))
```

## 🧩 사용된 Python 개념

- `defaultdict(list)` : 알파벳별 인덱스 수집
- `sorted(dict.keys())` : 알파벳 처리 순서 고정
- 2차원 DP 배열 : `dp = [[0]*2 for _ in range(L)]`
- 절댓값 이동 거리 계산 : `abs(a-b)`
- 최소 비용 전이 : `min(이전 상태 2개 + 현재 비용)`

- 시간복잡도 : `O(N + U log U)` (`N`=문자열 길이, `U`=등장한 알파벳 수)
- 공간복잡도 : `O(N + U)`

## ⚠️ 오답 포인트

- 문자 하나씩 처리하면 상태가 과도하게 커져 복잡해진다.
- `visited` 같은 BFS 관성으로 접근하면 핵심 상태 전이를 놓치기 쉽다.
- 직접 문자열을 지우는 시뮬레이션을 넣으면 구현만 복잡해지고 DP 본질에서 벗어난다.
- 이전 단계의 실제 끝 위치(`prev_left/prev_right`)를 잘못 잡으면 점화식이 즉시 틀어진다.
