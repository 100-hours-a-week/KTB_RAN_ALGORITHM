## 🧩 문제

* **문제 제목** : 돌 게임
* **문제 레벨** : 실버 5
* **문제 유형** : DP (Bottom-up), 게임 이론
* **문제 제공** : BOJ 9655
* **문제 링크** : https://www.acmicpc.net/problem/9655
* **코드 파일** : [stone-game-9655.js](../../algorithm-JS/DP/stone-game-9655.js)

---

## 🧩 문제 설명

돌 `N`개가 있고, 두 사람은 번갈아 돌을 가져간다.  
한 번에 가져갈 수 있는 돌 개수는 `1`개 또는 `3`개이다.  
마지막 돌을 가져가는 사람이 승리하며, 상근(SK)이 먼저 시작한다.

---

## 🧩 문제 핵심 포인트

* `dp[i]`를 "`i`개에서 현재 턴 플레이어가 이길 수 있는가"로 정의한다.
* 현재 상태에서 `1`개 또는 `3`개를 가져간 뒤 상대를 패배 상태로 만들 수 있으면 현재는 승리 상태다.
* 점화식:
  * `dp[i] = !dp[i-1] || !dp[i-3]`

---

## 🧩 내 풀이 방식

* 작은 상태부터 채우는 Bottom-up DP를 사용했다.
* 기저값:
  * `dp[1] = true` (1개를 바로 가져가 승리)
  * `dp[2] = false` (1개만 가져가면 상대가 마지막 1개를 가져감)
  * `dp[3] = true` (3개를 바로 가져가 승리)
* `i=4`부터 `n`까지 점화식으로 채운 뒤 `dp[n]` 결과에 따라 `SK`/`CY`를 출력했다.

---

## 🧩 코드 로직 정리

### 전체 흐름

```txt
1. n 입력
2. dp[i] = i개에서 현재 턴 승리 가능 여부로 정의
3. 기저값 dp[1], dp[2], dp[3] 설정
4. i=4..n 순회하며 dp[i] = !dp[i-1] || !dp[i-3]
5. dp[n]이 true면 SK, false면 CY 출력
```

---

## 🧩 코드 구현 (내 풀이)

```js
const fs = require("fs"); // /dev/stdin
const input = fs.readFileSync("../예제.txt").toString().trim().split("\n");

// 백준 제출 시
// const fs = require("fs");
// const input = fs.readFileSync(0, "utf-8").trim().split("\n");

const n = Number(input[0]);
const dp = Array(n + 1).fill(false);

// dp[i]: 돌이 i개 남았을 때 현재 턴 플레이어가 이길 수 있으면 true
dp[1] = true;
if (n >= 2) dp[2] = false;
if (n >= 3) dp[3] = true;

for (let i = 4; i <= n; i++) {
  dp[i] = !dp[i - 1] || !dp[i - 3];
}

console.log(dp[n] ? "SK" : "CY");
```

---

## 🧩 사용된 JS 개념

* `Array(n + 1).fill(false)` : 불리언 DP 배열 초기화
* 논리 연산자 `||`, `!` : 상대를 패배 상태로 보낼 수 있는지 판단
* 조건문(`if (n >= k)`) : 작은 입력에서 기저값 안전하게 처리

## ⚠️ 오답 포인트

* `for (let i = 4; i < n; i++)`로 두면 `dp[n]`이 계산되지 않아 오답이 난다. (`<= n` 필요)
* `SK`, `CY`를 문자열이 아닌 식별자로 쓰면 `ReferenceError`가 발생한다.
* `dp[2]`, `dp[3]` 기저값을 빼먹으면 점화식 전파가 틀어질 수 있다.
