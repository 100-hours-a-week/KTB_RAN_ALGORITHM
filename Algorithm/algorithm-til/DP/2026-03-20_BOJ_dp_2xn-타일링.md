## 🧩 문제

* **문제 제목** : 2xn 타일링
* **문제 레벨** : 실버 3
* **문제 유형** : DP (Bottom-up), 경우의 수
* **문제 제공** : BOJ 11726
* **문제 링크** : https://www.acmicpc.net/problem/11726
* **코드 파일** : [2xn-tiling-11726.py](../../algorithm-Python/DP/2xn-tiling-11726.py)

---

## 🧩 문제 설명

`2×n` 직사각형을 `1×2`, `2×1` 타일로 채우는 방법의 수를 구하는 문제다.  
정답은 `10007`로 나눈 나머지를 출력한다.

---

## 🧩 문제 핵심 포인트

* `dp[i]`를 "`2×i` 보드를 채우는 경우의 수"로 정의한다.
* `2×i`를 완성할 때 마지막 배치 방법은 딱 2가지다.
  * 마지막에 세로 타일 1개를 두는 경우
  * 마지막에 가로 타일 2개를 두는 경우
* 점화식:
  * `dp[i] = dp[i-1] + dp[i-2]`

---

## 🧩 내 풀이 방식

* 기저값을 먼저 고정한다.
  * `dp[1] = 1`, `dp[2] = 2`
* `i=3`부터 `n`까지 점화식으로 채운다.
* 각 단계에서 `10007` 모듈러를 적용해 숫자 크기를 관리한다.

---

## 🧩 코드 로직 정리

### 마지막 모양으로 상태를 결정

경우 1) 마지막이 세로 1개

```txt
| | | |
| | | |
      ↑
```

* 마지막 한 칸을 세로로 채운다.
* 그 앞은 `2×(i-1)`이 완성되어 있어야 한다.
* 경우의 수: `dp[i-1]`

경우 2) 마지막이 가로 2개

```txt
| | — —
| | — —
```

* 마지막 두 칸을 가로로 채운다.
* 그 앞은 `2×(i-2)`가 완성되어 있어야 한다.
* 경우의 수: `dp[i-2]`

따라서 결론:

* `dp[i] = dp[i-1] + dp[i-2]`
* 특히 `dp[3] = dp[2] + dp[1]`

핵심 차이:

* "아무 `2×2`, `2×1`에서 확장"이 아니라,
* "`2×i`의 마지막 모양을 정하면 직전 상태(`i-1` 또는 `i-2`)가 강제된다."

---

## 🧩 코드 구현 (내 풀이)

```python
import sys

# 로컬 테스트 시 사용
# sys.stdin = open("../예제.txt", "r")
input = sys.stdin.readline

MOD = 10007


def solution():
    n = int(input().strip())

    if n == 1:
        return 1
    if n == 2:
        return 2

    # dp[i] = 2xi 보드를 채우는 경우의 수
    dp = [0] * (n + 1)
    dp[1], dp[2] = 1, 2

    for i in range(3, n + 1):
        # 마지막 모양 기준 분기:
        # 1) 세로 1개로 끝나면 dp[i-1]
        # 2) 가로 2개로 끝나면 dp[i-2]
        dp[i] = (dp[i - 1] + dp[i - 2]) % MOD

    return dp[n]


print(solution())
```

## 🧩 사용된 Python 개념

* `list` 초기화: `dp = [0] * (n + 1)`
* 반복문: `for i in range(3, n + 1)`
* 모듈러 연산: `(a + b) % 10007`
* 빠른 입력: `sys.stdin.readline`

## ⚠️ 오답 포인트

* `dp[i]` 의미를 "최소 타일 개수"로 정의하면 점화식이 어긋난다.
* `dp[1]`, `dp[2]` 기저값을 틀리면 전체 결과가 틀어진다.
* 모듈러를 마지막에만 처리해도 정답은 되지만, 중간에 적용하는 습관이 안전하다.
