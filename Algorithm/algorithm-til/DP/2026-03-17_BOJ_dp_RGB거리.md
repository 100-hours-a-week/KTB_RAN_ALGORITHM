## 🧩 문제

- **문제 제목** : RGB거리
- **문제 레벨** : 실버 1
- **문제 유형** : DP (Bottom-up), 2차원 상태 전이
- **문제 제공** : BOJ 1149
- **문제 링크** : https://www.acmicpc.net/problem/1149
- **코드 파일** : [rgb-street-1149.js](../../algorithm-JS/DP/rgb-street-1149.js)
- **파이썬 코드 파일** : [rgb-street-1149.py](../../algorithm-Python/DP/rgb-street-1149.py)

---

## 🧩 문제 설명

N개의 집을 빨강/초록/파랑 중 하나로 칠할 때, 인접한 집의 색이 같지 않도록 하면서 전체 비용의 최솟값을 구하는 문제다.

---

## 🧩 문제 핵심 포인트

- `dp[i][c]`를 "`i`번째 집을 `c`색으로 칠했을 때의 최소 비용"으로 정의하면 조건이 자연스럽게 반영된다.
- 현재 집의 색이 정해지면 이전 집은 나머지 2색만 가능하다.
- 점화식:
  - `dp[i][0] = min(dp[i-1][1], dp[i-1][2]) + color[i][0]`
  - `dp[i][1] = min(dp[i-1][0], dp[i-1][2]) + color[i][1]`
  - `dp[i][2] = min(dp[i-1][0], dp[i-1][1]) + color[i][2]`

---

## 🧩 내 풀이 방식

- DP 정의를 먼저 고정했다: **i번째 특정 색상일 때 최소 비용**.
- 현재 상태를 먼저 본다:
  - 예) **i번째 집을 빨강으로 칠했을 때 최소 비용**을 명확한 문장으로 고정한다.
- 이전 상태는 항상 직전(`i-1`)만 본다:
  - 최솟값 문제이므로 직전 가능한 상태 2개 중 `min`을 고른다.
- 인사이트:
  - 최솟값 DP는 **(과거 중 최솟값) + 현재(필수 비용)** 구조로 쌓인다.

---

## 🧩 코드 로직 정리

### 전체 흐름

```txt
1. n, colorArr 입력
2. dp[n][3] 배열 생성
3. 첫 집 비용으로 dp[0][0], dp[0][1], dp[0][2] 초기화
4. i=1..n-1 순회
5. 각 색상별로 직전 2개 색 중 최소 + 현재 색 비용으로 갱신
6. 마지막 집의 3가지 색 비용 중 최소 반환
```

## 🧩 코드 구현 (내 풀이)

```js
// RGB거리에는 집이 N개 있다. 거리는 선분으로 나타낼 수 있고, 1번 집부터 N번 집이 순서대로 있다.

// 집은 빨강, 초록, 파랑 중 하나의 색으로 칠해야 한다.
// 각각의 집을 빨강, 초록, 파랑으로 칠하는 비용이 주어졌을 때, 아래 규칙을 만족하면서 모든 집을 칠하는 비용의 최솟값을 구해보자.

// 1번 집의 색은 2번 집의 색과 같지 않아야 한다.
// N번 집의 색은 N-1번 집의 색과 같지 않아야 한다.
// i(2 ≤ i ≤ N-1)번 집의 색은 i-1번, i+1번 집의 색과 같지 않아야 한다.

//1. DP 정의하기 : i가 ()였을때 집 칠하는 비용의 최솟값

//dp[][0] = 빨강 dp[][1] = 초록 dp[][2] = 파랑
//2. i번째 내가 할수있는 선택
//dp[i][0] = i번째 집을 빨강으로 칠했을때 최소비용 => dist의 역할을 함.
//최솟값이니까 min이겠군아.
//dp[i][1] = i번째 집을 초록으로 칠했을때 최소비용

//DP 매일 풀어야할 것같음. 매일 한문제
//3. 이전의 상태
//DP[i-1] = 파랑이었을경우
//1. DP[i][0] = Math.min(DP[i-1][1],DP[i-1][2]) + 현재빨강
//2. DP[i] = Math.min(DP[i-1][0],DP[i-1][1]) + 현재파랑
//3. DP[i] = Math.min(DP[i-1][0],DP[i-1][2]) + 현재초록
//DP[i] == DP[i-1]가 선택한 값만 아니면돼

//인사이트 : 최솟값을 구할때는 과거 + 현재 구조이다.

const fs = require("fs"); // /dev/stdin
const input = fs.readFileSync("../예제.txt").toString().trim().split("\n");

//백준 제출 시
// const fs = require("fs");
// const input = fs.readFileSync(0, "utf-8").trim().split("\n");

const n = Number(input[0]);
const colorArr = input.slice(1).map((i) => i.split(" ").map(Number));

function solution() {
  const dp = Array.from({ length: n }, () => Array(3).fill(0));

  //작은 문제로 큰문제를 푼다.
  dp[0][0] = colorArr[0][0];
  dp[0][1] = colorArr[0][1];
  dp[0][2] = colorArr[0][2];

  for (let i = 1; i < n; i++) {
    dp[i][0] = Math.min(dp[i - 1][1], dp[i - 1][2]) + colorArr[i][0];
    dp[i][1] = Math.min(dp[i - 1][0], dp[i - 1][2]) + colorArr[i][1];
    dp[i][2] = Math.min(dp[i - 1][0], dp[i - 1][1]) + colorArr[i][2];
  }

  return Math.min(...dp[n - 1]);
}

console.log(solution());
```

## 🧩 코드 구현 (파이썬 동일 풀이)

```python
import sys

sys.stdin = open("../예제.txt")


input = sys.stdin.readline
n = int(input())

colorArr = [list(map(int, input().split(" "))) for _ in range(n)]

def solution():
    dp = [[0]*3 for _ in range(n)];

    dp[0][0] = colorArr[0][0];
    dp[0][1] = colorArr[0][1];
    dp[0][2] = colorArr[0][2];

    for i in range(1,n) :
        dp[i][0] = min(dp[i-1][1],dp[i-1][2]) + colorArr[i][0];
        dp[i][1] = min(dp[i-1][0],dp[i-1][2]) + colorArr[i][1];
        dp[i][2] = min(dp[i-1][0],dp[i-1][1]) + colorArr[i][2];

    return min(dp[n-1])

print(solution());
```

## 🧩 사용된 JS 개념

- `Array.from({ length: n }, () => Array(3).fill(0))` : 2차원 DP 배열 초기화
- `Math.min()` : 가능한 이전 상태 중 최소 비용 선택
- 전개 연산자 `...` : 마지막 행 최솟값 계산(`Math.min(...dp[n - 1])`)
- `map(Number)` : 문자열 입력을 숫자 배열로 변환

## ⚠️ 오답 포인트

- `dp[i]`를 1차원으로만 두면 "현재 색상 상태"가 사라져 점화식이 꼬인다.
- `dp[0]` 초기값(첫 집 3색 비용)을 빼먹으면 모든 전이가 잘못된다.
- 현재 색과 같은 색을 이전 상태에 포함하면 문제 조건을 위반한다.
- 정답을 `dp[n-1][0..2]` 최소값으로 뽑지 않고 특정 색 하나만 반환하면 오답이 난다.

## 📝 회고 KPT

### 🟢 Keep

- DP를 풀 때 상태를 한 문장으로 정의하고 시작한 점이 좋았다.
- 현재 상태를 먼저 고정한 뒤 직전 상태를 붙이는 흐름이 안정적이었다.
- 최솟값 문제를 `min(과거) + 현재` 구조로 일관되게 유지했다.

### 🔴 Problem

- 초기에 `dp[i]` 의미를 흐리게 잡으면 색상 충돌 조건을 놓치기 쉽다.
- 직전 상태를 넘어서 더 먼 과거를 직접 참조하려고 하면 식이 불필요하게 복잡해진다.
- 마지막 반환에서 3색 최소를 잊는 실수가 발생할 수 있다.

### 🟡 Try

- 다음 DP 문제도 먼저 "`dp[i][state]` 한 문장 정의"를 체크리스트로 고정한다.
- 전이식 작성 전 "현재 상태에서 가능한 직전 상태"를 1분 내로 표로 정리한다.
- 제출 전 마지막 줄은 항상 "`return min(마지막 상태들)`인지" 점검한다.
