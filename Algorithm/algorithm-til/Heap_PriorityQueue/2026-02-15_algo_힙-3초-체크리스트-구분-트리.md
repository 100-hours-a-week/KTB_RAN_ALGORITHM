# 힙 3초 체크리스트 + 자료구조 선택 트리

코테 실전에서 3초 안에 "힙을 써야 하는 문제인지" 판별하기 위한 정리.

---

## 힙 문제 3초 체크리스트

### 1) "가장 작은/큰 값을 계속 꺼낸다"

아래 표현이 보이면 힙 우선 검토:

- 가장 작은 두 개
- 가장 큰 하나
- 최소값/최대값 반복
- 가장 먼저 끝나는 작업
- 가장 오래 걸리는 작업

### 2) 매번 정렬해야 할 것 같은 구조

이 패턴이 보이면:

```txt
정렬 -> 뽑기 -> 다시 정렬 -> 뽑기
```

- 정렬 반복은 비용이 큼
- 힙으로 바꾸면 추출/삽입 비용을 줄일 수 있음

### 3) 실시간 데이터에서 최적값 유지

- 현재까지 중 가장 작은 K개
- 가장 큰 값 유지
- Top-K
- 스트리밍 데이터

### 4) 작업 스케줄링

- 디스크 컨트롤러
- CPU 스케줄링
- 작업 처리 순서 최적화

### 5) 조건 초과 시 제거/면제

- K번까지 제거 가능
- R번 무료 처리 가능
- 초과 시 하나 제거

### 6) 입력이 매우 큼

- `n >= 100,000`
- 정렬 반복이 위험한 상황

---

## 힙 문제의 공통 구조

```txt
1. 값 넣기
2. 최소/최대 꺼내기
3. 조건 체크
4. 다시 넣기
```

이 패턴이면 힙 가능성이 높다.

---

## 힙 vs 정렬 vs 투포인터 구분 트리

### 1) 한 번 정렬하면 끝나는가?

- YES -> 정렬
- NO -> 다음 질문

### 2) 최소/최대를 반복 추출해야 하는가?

- YES -> 힙
- NO -> 다음 질문

### 3) 연속 구간(부분 배열) 문제인가?

- YES -> 투포인터(슬라이딩 윈도우)
- NO -> 다음 질문

### 4) 두 배열 매칭/짝짓기 문제인가?

- 정렬 후 한 번 순회로 해결 가능 -> 정렬 + 투포인터

### 5) 조건 초과 시 하나 제거하는 구조인가?

- YES -> 힙

---

## 시각적 요약

```txt
문제 읽기
   |
   |- 한 번 정렬 후 끝? -> YES: 정렬
   |- 최소/최대 반복 추출? -> YES: 힙
   |- 연속 구간? -> YES: 투포인터
   |- 두 배열 매칭? -> YES: 정렬 + 투포인터
   |- 조건 초과 시 제거? -> YES: 힙
```

## 실전 예시

| 문제 | 자료구조/알고리즘 |
| --- | --- |
| 더 맵게 | 힙 |
| 디스크 컨트롤러 | 힙 |
| 두 큐 합 같게 만들기 | 투포인터 |
| 숫자 게임 | 정렬 |
| 입국 심사 | 이분 탐색 |

## 한 줄 기억법

> 최소/최대 반복 -> 힙  
> 한 번 정렬 -> 정렬  
> 연속 구간 -> 투포인터
